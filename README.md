# TelegramSupportBot

TelegramSupportBot — простой Telegram-бот поддержки на **Python** и **Aiogram 3**. Пользователь пишет в бота, бот собирает сообщения в тикет, а администратор получает уведомление, открывает тикет в панели, отвечает пользователю и закрывает обращение.

## Возможности

- **Тикеты по каждому обращению**: новый открытый тикет создаётся автоматически, когда пользователь пишет в поддержку.
- **История сообщений**: сообщения пользователя и администратора сохраняются в JSON-файлах.
- **Задержка подтверждения**: бот ждёт паузу после последнего сообщения пользователя и только потом отправляет подтверждение и уведомление админу.
- **Админ-панель**: кнопка «Панель» показывает открытые тикеты, даёт открыть карточку тикета, ответить или закрыть обращение.
- **Защита от очень длинных сообщений**: карточка тикета режется на несколько Telegram-сообщений, поэтому даже огромный текст не ломает открытие тикета.
- **Адаптивная обработка контента**: текст, подписи, голосовые, фото, видео, видео-кружки, документы, аудио, GIF/анимации, стикеры, локации, места, контакты, опросы и dice-сообщения сохраняются в читаемом виде.

## Как это работает

1. Пользователь отправляет сообщение боту.
2. Если у пользователя нет открытого тикета, создаётся тикет с ID вида `NR-XXXXXX`.
3. Сообщение добавляется в историю тикета.
4. После паузы `ACK_DELAY` бот подтверждает пользователю получение заявки и уведомляет администратора.
5. Администратор открывает «Панель», выбирает тикет, читает последние сообщения, отвечает или закрывает тикет.
6. После закрытия пользователь может написать снова — будет создан новый тикет.

## Поддерживаемые типы сообщений

| Тип | Как отображается в тикете |
| --- | --- |
| Текст | Оригинальный текст сообщения |
| Медиа с подписью | Тип медиа, `file_id` и подпись |
| Голосовое | `file_id` и длительность |
| Фото | `file_id` лучшего варианта фото |
| Видео / видео-кружок | `file_id` и длительность |
| Документ | имя файла и `file_id` |
| Аудио | название, `file_id` и длительность |
| GIF / анимация | `file_id` |
| Стикер | emoji и `file_id` |
| Локация / место | координаты и ссылка на Google Maps |
| Контакт | имя и телефон |
| Опрос | вопрос и варианты ответа |
| Dice | emoji и выпавшее значение |
| Неизвестный тип | техническое имя типа сообщения |

> Важно: бот сохраняет `file_id`, чтобы администратор видел, что именно прислал пользователь. Файлы не скачиваются на сервер.

## Структура проекта

```text
bot/
  core/
    dispatcher.py    # регистрация обработчиков
    json_storage.py  # JSON-хранилище с блокировкой
    loader.py        # Bot и Dispatcher
    utils.py         # безопасная отправка, длинные сообщения, форматирование входящих сообщений
  handlers/
    admin.py         # админ-панель, ответы, закрытие тикетов
    user.py          # входящие сообщения пользователей
  services/
    scheduler.py     # отложенное подтверждение и уведомление админа
    tickets.py       # создание тикетов и история сообщений
    users.py         # состояние пользователей
config.py            # настройки из переменных окружения
example.env          # пример конфигурации
requrements.txt      # зависимости проекта
```

## Быстрый старт

### 1. Установите зависимости

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requrements.txt
```

### 2. Настройте окружение

Скопируйте пример конфигурации и заполните значения:

```bash
cp example.env .env
```

Переменные:

| Переменная | Описание | Пример |
| --- | --- | --- |
| `BOT_TOKEN` | токен Telegram-бота от BotFather | `123456:ABC...` |
| `ADMIN_ID` | Telegram ID администратора | `123456789` |
| `ACK_DELAY` | пауза перед подтверждением и уведомлением, сек. | `30` |
| `USERS_PATH` | путь к JSON-файлу пользователей | `bot/data/users.json` |
| `TICKETS_PATH` | путь к JSON-файлу тикетов | `bot/data/tickets.json` |
| `MESSAGES_PATH` | папка с историями сообщений | `bot/data/messages` |

### 3. Запустите бота

```bash
python3 bot/main.py
```

## Администрирование

- Напишите `/start` от имени администратора — бот покажет нижнюю кнопку «Панель».
- Нажмите «Панель», чтобы увидеть открытые тикеты.
- Откройте тикет, чтобы посмотреть последние сообщения.
- Нажмите «Ответить 📩», отправьте текст ответа — бот перешлёт его пользователю.
- Нажмите «Закрыть ❌», чтобы закрыть тикет и разрешить пользователю создать новый.

## Хранение данных

Проект использует JSON-файлы, поэтому его легко запускать без базы данных:

- `tickets.json` хранит список тикетов и их статусы.
- `users.json` хранит состояние пользователей.
- `messages/ticket_<ID>.json` хранит историю конкретного тикета.

Для небольшого саппорта этого достаточно. Если нагрузка вырастет, JSON-хранилище лучше заменить на SQLite/PostgreSQL.

## Проверка перед запуском

Минимальная проверка синтаксиса:

```bash
python3 -m py_compile bot/main.py bot/core/*.py bot/handlers/*.py bot/services/*.py config.py
```
